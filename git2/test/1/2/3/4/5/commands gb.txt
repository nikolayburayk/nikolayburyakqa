1)ls - Просмотр содержимого, показывает текущую папку.
Если в параметрах указать путь, то она перечислит содержимое конечного каталога.
-l -a Первая форматирует вывод в виде списка с более 
подробной информацией, а вторая включает показ скрытых файлов.

2)cat - читает данные из файла или стандратного ввода и выводит	их на экран

3)cd Music/(интересующая папка) - позволяет перейти в указанный каталог

4)pwd - показывает путь где мы находимся сейчас

5)cd .. - возвращает на 1 шаг назад по папкам

6)cd ../.. - возвращает на 2 шага назад по папкам

7)cd ~ - возвращает в коренную папку

8)mkdir "name" - создание папки, так же можно создать папку в другом
каталоге mkdir "music/папка". 
mkdir {папка,папка,папка} - создаем сразу три папки
Более сложная задача создать три папки в 
каталоге mkdir music/папка/{папка,папка,папка}
mkdir -p ~/

9)file - позволяет узнать тип данных содержащихся в документе.
Будет не лишним проверить незнакомый файл через данную команду.

10)cp - позволяет полностью копировать файлы из директории.
cp->опции->файл-источник->файл-приемник or 
cp->опции->файл-источник->директория-приемник/ - слэш в конце позволяет записать
файл с оригинальным именем в директорию
$ cp c:/Users/Nikolay/git2/photo.jpg c:/Users/Nikolay/pictures/
копирование папок осуществляется с ключом -r
cp -r c:/Users/Nikolay/videos c:/Users/Nikolay/git2/
cp ~/test* ~/Documents/ - скопировать все файлы содержащие "test"

11)mv - предназначена для перемещения файлов или переименовывания, если выполняется
в одной папке.
создать файл в директории touch file
mv file 222 - называть файл другим именем(222)
mv ~/git2/222 ~/git2/333 - перенос файла в этой же папке но уже с другим названием.


12)rename 's\.htm/\.html/' *.htm - переименовать все htm файлы в html

13)git add - добавление файла\папки в stage(форма подготовки к отправлению) для 
отправки в репозиторий

14)git commit -m "commit's name" - выбираем название комита для отправки на
репозиторий в хаб.

15) git push - всё что подготовили до этого, отправляем данной командой на "верх" в 
репозиторий

16) git status - находясь в папке команда покажет в каком состоянии находятся файлы 
перед комитом. Выдаёт подсказку после написания команды.
